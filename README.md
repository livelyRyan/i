# Resume Workflow

This repo keeps `resume.md` as the single source of truth for the resume.

## Source of truth

- `resume.md`: primary resume content
- `index.html`: generated by `scripts/render_resume.py` from `resume.md`; committed for **Deploy from a branch** (`master` / `/ (root)`). Pushes to `master` also run **GitHub Actions** to build `_site` and deploy when Pages source is **GitHub Actions** (see `.github/workflows/render-resume-pages.yml`).
- `resume-profile.jpg`: used by the renderer for the site hero photo (not required inside `resume.md` itself).
- `代表作/`: portfolio HTML and static assets (same paths on the published site, e.g. `代表作/...`)

## 简历与个人站维护规范

以下约定原先写在 `RESUME_WORKFLOW.md`，与当前仓库实现对齐后收拢到此文件，避免「文档描述一套、仓库另一套」。

### 原则

1. **`resume.md` 是唯一真源**：投递与网站首页共用同一套正文，不单独再维护一份「投递版」经历段落。
2. **投递/ATS 向**：正文以单栏、可复制的 Markdown 为主；避免在 `resume.md` 里嵌表格、HTML、复杂排版技巧。
3. **网站可更丰富**：细节放在 `代表作/` 等独立页面；首页由渲染脚本对齐 `resume.md`，不在站点里复制一整套经历 bullet 以免分叉。
4. **LLM 修改有边界**：优先只改标记区块内的文字，不擅自改公司、时间等事实字段。

### `resume.md` 建议格式（给人工与 LLM）

- **允许的语法**：标题、段落、`-` 列表、行内链接为主。
- **章节顺序**：建议保持 `# 姓名` → `## Summary` → `## Experience` → `## Skills` → 其余（如 `## Projects`、`## Education` 等）；具体标题以当前文件为准。
- **日期**：建议统一为 `YYYY-MM — YYYY-MM` 或「至今 / Present」等与全文一致的风格。
- **Experience**：职位、公司、时间与要点层次清晰；要点优先写结果与可量化影响。
- **Skills**：可按分组列表（如 Languages / LLM / Frameworks / Infrastructure 等），便于以后出英文版。

### LLM 可编辑区块

在 `<!-- LLM:BEGIN … -->` 与 `<!-- LLM:END … -->` 之间修改（例如 `summary`、`skills`）。不要改动区块外的标题结构或事实性字段，除非用户明确提供新信息。输出整文件或补丁均可。

### 当前仓库「已落地」 vs 「未在此仓库」

| 项目 | 状态 |
| --- | --- |
| Markdown 真源 + Python 渲染 `index.html` / Actions 部署 | 已落地 |
| `代表作/` 项目页 | 已落地 |
| `content/projects` 等独立 Markdown 内容库、Astro 工程 | **未**在本仓库；若以后引入，再扩展文档即可 |
| `dist/resume.pdf`、`dist/resume.docx`、Pandoc 一键导出 | **未**在本仓库；需要时可另加脚本 |

## 如何更新网站（GitHub Pages）

打开 **Settings → Pages → Build and deployment**：

- **GitHub Actions**（推荐）：Source 选 **GitHub Actions**，保存后每次推 `master` 会跑「Render Resume Pages」：**build** 上传产物，**deploy** 发布。若长期 404，到 **Actions** 打开最近一次运行，确认 `deploy` 成功且无 Environment 审批阻塞。
- **Deploy from a branch**（备用）：Source 改为 **Deploy from a branch**，Branch **master**，Folder **`/ (root)`**。仓库根目录必须有已提交的 **`index.html`**（否则站点根路径会 404）。

两种源**不要混用**：选 Actions 时以运行产物为准；选 branch 时以仓库里 `index.html` 为准。可选在改完 `resume.md` 后本地渲染并一并提交 `index.html`，与 branch 源一致：

```bash
python3 scripts/render_resume.py --in resume.md --out index.html
git add resume.md index.html 代表作/
git commit -m "docs: AI update resume and portfolio"
git push origin master
```

推送后，Actions 工作流会重新渲染并部署（若已启用 Actions 源）。
